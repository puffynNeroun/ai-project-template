Local-first workflow for AI-assisted development

See how Forge turns AI work into a verifiable delivery flow.

Forge does not replace your AI tools. It wraps them in task contracts, artifacts, checks, review, and completion evidence.

Current step

Define Task

Forge starts by turning an idea into a scoped task contract.

Lifecycle

From idea to completed task

Each step creates or checks evidence. The goal is to avoid chaotic copy-paste development and keep AI-assisted work reviewable.

Under the hood

What Forge records

Forge makes AI-assisted delivery auditable by storing task state and role evidence as files in the repository.

Task contract

.forge/tasks/TASK-XXXX.yaml

Defines scope, allowed files, protected files, acceptance criteria, and required checks.

Plan artifact

.forge/artifacts/TASK-XXXX/plan-001.md

Records how the work should be done before implementation begins.

Build report

.forge/artifacts/TASK-XXXX/build-report-001.md

Explains what changed and what is ready for testing.

Test report

.forge/artifacts/TASK-XXXX/test-report-001.md

Records verification, checks, and test evidence.

Review report

.forge/artifacts/TASK-XXXX/review-report-001.md

Confirms whether the work is acceptable before PR.

Why it matters

AI can help, but Forge keeps the process controlled.

Scope before code

The task contract defines what is allowed before implementation starts.

Evidence at each stage

Planning, building, testing, and review leave artifacts behind.

Checks before merge

Forge status, smoke tests, verify, and CI keep the repo honest.

What this demo does not claim

This visual demo is static. It does not mean Forge has an autonomous agent runner, SaaS dashboard, npm publishing, Telegram approvals, automatic PR creation, or automatic code generation.
