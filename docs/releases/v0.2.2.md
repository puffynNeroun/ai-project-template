# Forge v0.2.2 Release Notes

## Status

These notes prepare Forge v0.2.2.

This document does not create the git tag or the GitHub release. Tagging and GitHub release creation should happen in a later release task after these notes are merged.

## Summary

Forge v0.2.2 is a public presentation and usability release.

After v0.2.1, Forge gained three important improvements:

- a packaged Forge CLI entrypoint;
- a clearer open-source quality README;
- an end-to-end demo scenario.

The main result is simple:

    v0.2.2 makes Forge easier to use and easier to explain.

This release notes task only documents the release. It does not change runtime behavior, CLI behavior, validator behavior, workflow contracts, or role contracts.

## What's Included

v0.2.2 summarizes the work from:

- TASK-0020 — Package normal Forge CLI
- TASK-0021 — Rewrite README for portfolio/open-source quality
- TASK-0022 — Add Forge demo scenario

Together, these tasks make Forge more understandable as a local-first workflow toolkit for safe AI-assisted software delivery.

## TASK-0020 — Packaged Forge CLI

TASK-0020 added a packaged `forge` bin entrypoint for a cleaner command shape.

The concise CLI form includes:

    forge status
    forge smoke
    forge verify
    forge task new
    forge task stage
    forge task complete
    forge artifact new

The existing pnpm commands remain supported and are still the most explicit direct commands for this repository:

    pnpm -C tools/forge-validator run status
    pnpm -C tools/forge-validator run workflow:smoke
    pnpm -C tools/forge-validator verify

Accuracy note:

Forge exposes a package bin entrypoint, but this release does not claim public npm publishing or global installation by default.

## TASK-0021 — README Open-Source Quality

TASK-0021 rewrote the root README so a first-time GitHub visitor can understand Forge more quickly.

The README now explains:

- what Forge is;
- why Forge exists;
- the problem of chaotic AI-assisted development;
- the Forge lifecycle;
- task contracts;
- artifacts as evidence;
- safety controls;
- quickstart commands;
- the packaged CLI shape;
- current project status;
- roadmap;
- what Forge is not.

The README positions Forge as the workflow layer around AI-assisted development, not as a replacement for ChatGPT, Cursor, Codex, Aider, OpenHands, or engineering judgment.

## TASK-0022 — Demo Scenario

TASK-0022 added:

    docs/DEMO.md

The demo walks through a small realistic documentation task:

    Add a short FAQ section to a product README.

The demo shows the lifecycle from task definition through completion:

- define the task;
- write the plan;
- build the change;
- test the result;
- review the result;
- open and merge the implementation PR;
- complete the task after merge.

It also explains the evidence chain:

    .forge/tasks/TASK-XXXX.yaml
    .forge/artifacts/TASK-XXXX/plan-001.md
    .forge/artifacts/TASK-XXXX/build-report-001.md
    .forge/artifacts/TASK-XXXX/test-report-001.md
    .forge/artifacts/TASK-XXXX/review-report-001.md

The demo makes Forge easier to evaluate because it shows how the workflow behaves in practice, not only in theory.
