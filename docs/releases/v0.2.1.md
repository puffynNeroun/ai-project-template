# Forge v0.2.1 Release Notes

## Status

Prepared release notes for the future Forge v0.2.1 patch release.

This document does not create the Git tag, GitHub release, or package publication.

## Summary

Forge v0.2.1 is a patch release after Forge v0.2.0.

Forge v0.2.0 established the controlled local lifecycle CLI.

Forge v0.2.1 documents how to use that stable template to start a real product repository.

## Main Change

Added project bootstrap documentation:

docs/BOOTSTRAP_PROJECT.md

The guide explains how to create a new product repository from this template and start work without losing the Forge workflow structure.

## Why This Release Exists

The v0.2.0 tag was created before the project bootstrap guide was added.

That means v0.2.0 contains the lifecycle CLI foundation, but not the bootstrap documentation.

v0.2.1 captures the next stable template state:

- Forge lifecycle CLI from v0.2.0
- project bootstrap guide for real products
- verified smoke workflow
- verified contract validation baseline

## Added Documentation

### Project Bootstrap Guide

docs/BOOTSTRAP_PROJECT.md explains:

- how to create a new repository from this template
- how to clone the new repository
- how to run the first verification commands
- how to rename and adapt project identity
- how to decide whether to keep or reset template task history
- how to define the first real product task
- how to run the Forge lifecycle
- how to work with Cursor, Codex, or another AI assistant
- how to bootstrap a Pixardia-style product
